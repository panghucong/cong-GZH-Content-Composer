# 快速模式详细流程

> **章节代码**：S1
> **适用场景**：用户需求明确，希望快速获得初稿
> **Token消耗**：~3,000字（仅加载此文件）

---

## 执行步骤

### 0. 智能默认值系统（新增）

**核心目的**：减少用户需要输入的参数，实现"一键式"快速生成。

**执行流程**：

1. **读取历史偏好**（从working memory）：
   ```
   读取 memory/MEMORY.md，查找以下信息：
   - 用户历史行业偏好
   - 用户历史目标用户
   - 用户历史内容类型
   - 用户历史排版偏好（色调/预设）
   - 用户历史写作风格偏好
   ```

2. **设置智能默认值**：
   | 参数 | 默认值来源 | 覆盖条件 |
   |------|----------|----------|
   | 行业 | 历史偏好 > 自动识别 > "通用" | 用户显式指定 |
   | 目标用户 | 历史偏好 > 行业Profile默认 > "通用" | 用户显式指定 |
   | 内容类型 | 自动推断 > 历史偏好 > "干货科普" | 用户显式指定 |
   | 字数档位 | 历史偏好 > "标准（800～1200字）" | 用户显式指定 |
   | 写作风格 | 历史偏好 > 行业Profile默认 > "行业默认" | 用户显式指定 |
   | 排版预设 | 历史偏好 > 行业默认 > P2（干货科普） | 进入排版环节时确认 |
   | 主色调 | 历史偏好 > 行业默认 > T1（浅蓝） | 进入排版环节时确认 |

3. **应用默认值**：
   - 用户说"写推文"/"生成推文"→ 直接使用所有默认值，生成内容
   - 用户说"写保险推文"→ 覆盖"行业"默认值，其他使用默认值
   - 用户说"写一篇给宝宝买保险的种草文"→ 覆盖"行业"+""内容类型"，其他使用默认值

**智能推断规则**（增强版）：

| 用户输入 | 推断结果 |
|---------|---------|
| "给宝宝买保险" | 行业=保险，目标用户=宝妈/宝爸，内容类型=产品种草 |
| "职场新人时间管理" | 行业=职场，目标用户=职场新人，内容类型=干货科普 |
| "重疾险vs医疗险" | 行业=保险，内容类型=产品种草/对比评测 |
| "最新医保政策解读" | 行业=医疗，内容类型=干货科普/热点评论 |
| "简单介绍一下" | 字数档位=短文（300～500字） |
| "详细讲讲" | 字数档位=长文（1500～2000字） |

### 1. 引导用户用一句话描述需求

**引导话术**：

> 请用工整的一两句话描述你想写的推文，例如：
> "我要写一篇面向职场新人的时间管理干货，重点是番茄工作法和任务优先级排序"
> 收到后我将直接生成初稿，不再逐步询问。

### 2. 自动解析 + 最多1次补充询问

用户回复后，AI自动解析其中隐含的信息：

| 解析字段 | 说明 |
|---------|------|
| 行业 | 从关键词识别（如"保险"→保险，"基金"→金融） |
| 目标用户 | 从描述推断（如"职场新人"→职场新人） |
| 内容类型 | 从动词推断（如"推荐"→产品种草，"科普"→干货科普） |
| 核心主题 | 提取核心关键词 |
| 字数档位 | 从"简短"→短文，"详细"→长文，默认→标准 |

**补充询问规则**：
- 能识别出 **行业+用户+类型+字数** 4个字段 → **不询问**，直接进入内容创作
- 能识别出 **行业+用户+核心主题** 3个字段 + 用户说"直接写" → **不询问**
- 缺失关键字段（如无法识别行业）→ **最多补充询问1次**（仅问最关键缺失项）

**行业感知**（G09）：解析用户输入时同步识别行业代码，动态加载对应行业 Profile。

### 3. 长文大纲确认（条件触发）

**触发条件**：解析出字数档位=长文（1500字以上）

**输出格式**：

> 已解析你的需求 🙌 由于是长文内容，先给你一个大纲确认方向：
>
> **【内容大纲】**
> - **标题方向**：[1句话概括核心话题]
> - **目标用户**：[自动识别的用户群体]
> - **写作框架**：[根据行业profile映射表选择的框架名，格式：F{ID}]
> - **核心结构**：
>   1. [开头引入：痛点/场景/悬念]
>   2. [第一个核心要点：...]
>   3. [第二个核心要点：...]
>   4. [第三个核心要点：...]
>   5. [结尾引导：...]
>
> 请确认大纲方向，或告诉我需要调整的部分。回复"直接写"则跳过确认。

### 4. 三视角写入（A1）

生成内容时融合三个视角：

| 视角 | 要点 |
|------|------|
| **专业视角** | 确保事实准确、逻辑严谨、数据可溯源 |
| **用户视角** | 确保场景真实、痛点精准、行动可执行 |
| **传播视角** | 确保钩子有力、节奏紧凑、分享友好 |

### 5. 钩子生成

根据行业 Profile 的钩子模板库（或 `profiles/{行业代码}.md`），为文章开头生成匹配的钩子。

**钩子类型**：

| 类型 | 示例 |
|------|------|
| 痛点型 | "你是不是也遇到过..." |
| 悬念型 | "90%的人都不知道..." |
| 数据型 | "最新数据显示..." |
| 故事型 | "上周我遇到一个客户..." |

### 6. 【C1】内容必填件校验（Quality Gate）

**核心目的**：避免初稿仅得8分、需要8轮补丁的根因问题。

**执行流程**：

1. 根据当前选择的 **写作框架 ID** + **内容类型**，从框架库的「框架必填件矩阵」中读取该组合的必填组件清单（见 `frameworks.md`）
2. 逐项检查每个必填组件是否已有素材/计划：
   - ✅ 已有 → 标记为"就绪"
   - ⚠️ 缺失但可推断 → AI 自动补充后标记为"自动补齐"
   - ❌ 缺失且无法推断 → 列入"待确认清单"
3. 输出简明校验报告：

> **📋 内容必填件校验报告**
>
> | 必填组件 | 状态 | 说明 |
> |----------|------|------|
> | {组件名} | ✅ 就绪 | |
> | {组件名} | ⚠️ 自动补齐 | {AI 如何补齐} |
> | {组件名} | ❌ 缺失 | 需你补充：{具体缺什么} |
>
> **通过率**：X/Y（Y=总必填件数）
> - 通过率 = 100% 或用户确认"继续" → 直接进入下一步
> - 存在 ❌ 项 → 先补充缺失项再生成（或用户说"跳过"则标记⚠️ 并继续）

### 7. 【C3】目标分数预检（可选）

若用户提到目标分数，进行预检提示：

> **🎯 质量预检**：你的目标分为{XX}分。根据当前框架+类型的经验数据，首次初稿平均约{XX-15~20}分，预计需要{1-3}轮优化可达标。

### 8. 直接生成推文初稿

生成内容包含：
- **3个备选标题**（悬念型/数字型/情绪型）
- **完整正文**（融合三视角+钩子）
- **结尾引导**（关注引导/互动提问/延伸阅读）

**同时进行评分**（按行业Profile的评分权重）。

### 9. 质量门控检查

**引用 shared-definitions.md 第十章**，不再独立定义。核心规则：
- 初稿低于80分自动重写，最多重写**2次**（即最多3稿）
- 最终稿必须达到92分以上才能输出
- 详细流程、决策树、安全限制见 `shared-definitions.md 第十章`

**迭代内容漂移检测**：每轮迭代后，对比当前内容与用户原始需求的关键词匹配度，匹配度下降超过20%自动回滚。

**自动重写章节映射规则**：

| 扣分最多子项 | 优先重写内容 | 重写策略 |
|-------------|-------------|----------|
| 事实准确性 | 涉及数据/事实的段落 | 核实数据、补充来源 |
| 标题吸引力 | 3个备选标题 | 调整标题公式 |
| 痛点匹配度 | 痛点引入部分 | 重写痛点描述 |
| 关键词覆盖 | 正文中段 | 补充核心关键词 |

## 快速通路判定表

| 条件 | 跳过动作 | 备注 |
|------|---------|------|
| 能识别出 **行业+用户+类型+字数** 4个字段 | 跳过引导确认，直接进入内容创作 | 最常见快速通路 |
| 能识别出 **行业+用户+核心主题** 3个字段 + 用户说"直接写" | 跳过引导，缺失项自动推断 | 次常见 |
| 历史记忆中有排版偏好 | 进入排版环节时跳过询问 | 仅影响排版环节 |
