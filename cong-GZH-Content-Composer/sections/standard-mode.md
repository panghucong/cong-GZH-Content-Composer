# 标准模式详细流程

> **章节代码**：S2
> **适用场景**：需要详细确认前提，确保内容精准匹配需求
> **Token消耗**：~3,500字（仅加载此文件）

---

## 0. 智能默认值系统（新增）

**核心目的**：减少用户需要输入的参数，实现"一键式"快速生成。

**执行流程**：

1. **读取历史偏好**（从working memory）：
   ```
   读取 memory/MEMORY.md，查找以下信息：
   - 用户历史行业偏好
   - 用户历史目标用户
   - 用户历史内容类型
   - 用户历史排版偏好（色调/预设）
   - 用户历史写作风格偏好
   ```

2. **设置智能默认值**：
   | 参数 | 默认值来源 | 覆盖条件 |
   |------|----------|----------|
   | 行业 | 历史偏好 > 自动识别 > "通用" | 用户显式指定 |
   | 目标用户 | 历史偏好 > 行业Profile默认 > "通用" | 用户显式指定 |
   | 内容类型 | 自动推断 > 历史偏好 > "干货科普" | 用户显式指定 |
   | 字数档位 | 历史偏好 > "标准（800～1200字）" | 用户显式指定 |
   | 写作风格 | 历史偏好 > 行业Profile默认 > "行业默认" | 用户显式指定 |
   | 参考资料 | 历史偏好 > 无 | 用户显式指定 |

3. **应用默认值**：
   - 在"引导话术"中，**预填默认值**（如"② **目标用户**：职场新人（历史偏好）"）
   - 用户可以直接回复"确认"或"开始写"，无需重新输入所有参数
   - 用户也可以修改任意一个参数

**智能推断规则**（增强版）：

| 用户输入 | 推断结果 |
|---------|---------|
| "给宝宝买保险" | 行业=保险，目标用户=宝妈/宝爸，内容类型=产品种草 |
| "职场新人时间管理" | 行业=职场，目标用户=职场新人，内容类型=干货科普 |
| "重疾险vs医疗险" | 行业=保险，内容类型=产品种草/对比评测 |
| "最新医保政策解读" | 行业=医疗，内容类型=干货科普/热点评论 |
| "简单介绍一下" | 字数档位=短文（300～500字） |
| "详细讲讲" | 字数档位=长文（1500～2000字） |

## 执行步骤

### 1. 解析用户回复

**解析规则**：

| 用户输入方式 | 解析方法 |
|-------------|---------|
| 一次性回答所有问题 | 按序号提取答案 |
| 分条确认 | 逐条解析并确认的字段 |
| 描述式回答 | 从描述中推断（如"给新手爸妈"→目标用户=宝妈/宝爸） |
| 说"开始写吧" | 缺失项自动推断 |

**自动推断规则**：

| 缺失字段 | 推断依据 |
|---------|---------|
| 行业 | 从描述关键词推断，无法推断则使用"通用" |
| 目标用户 | 从行业Profile的默认用户开始，或推断为"通用" |
| 内容类型 | 从动词推断（"推荐"→产品种草，"科普"→干货科普） |
| 字数档位 | 默认"标准（800～1200字）" |
| 写作风格 | 使用行业Profile的默认风格 |

### 2. 确认信息（可选）

若用户未明确确认，输出确认信息：

> 已收到你的信息 🙌 确认一下：
>
> - 行业/领域：{行业名}
> - 目标用户：{用户群体}
> - 内容类型：{类型名}
> - 字数档位：{档位名}
> - 写作风格：{风格名或"行业默认"}
> - 参考资料：{有/无}
>
> 确认无误请回复"开始写"，需要调整请告诉我。

### 3. 长文大纲确认（条件触发）

**触发条件**：字数档位=长文（1500字以上）

（同快速模式的大纲确认流程，见 `sections/quick-mode.md`）

### 4. 生成初稿

**生成要点**：

1. **三视角写入**（A1）：融合专业视角+用户视角+传播视角
2. **钩子生成**：根据行业Profile的钩子模板库生成开头钩子
3. **框架应用**：按内容类型×框架映射表选择框架（见 `shared-definitions.md 第六章`）
4. **必填件校验**（C1）：检查框架必填件清单（见 `frameworks.md`）
5. **生成内容**：含3个备选标题+完整正文+结尾引导
6. **评分**：按行业Profile的评分权重进行评分

## 与快速模式的区别

| 对比项 | 快速模式 | 标准模式 |
|-------|---------|---------|
| 引导方式 | 1句话描述需求 | 分步骤确认前提 |
| 询问次数 | 最多1次补充询问 | 可多次确认 |
| 适用场景 | 需求明确 | 需求模糊/需要精准匹配 |
| Token消耗 | 低（~3,000字） | 中（~3,500字） |
