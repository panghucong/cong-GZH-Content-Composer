# 批量生成模式详细流程

> **章节代码**：S4
> **适用场景**：需要一次性生成多篇推文
> **Token消耗**：~2,500字（仅加载此文件）

---

## 执行步骤

### 1. 引导用户输入主题列表

**引导话术**：

> 好的，进入批量生成模式 🙌
>
> 请输入你要生成的推文主题列表，每个主题一行，例如：
>
> 保险配置入门指南
> 重疾险vs医疗险，怎么选？
> 给宝宝买保险，这3点必须知道
>
> 也可以直接回复"自动选题"，我将根据行业热点生成主题列表。

**主题列表格式**：

| 格式 | 示例 |
|------|------|
| 每行一个主题 | "主题1\n主题2\n主题3" |
| 逗号分隔 | "主题1,主题2,主题3" |
| 数字编号 | "1.主题1 2.主题2 3.主题3" |

**自动选题**（条件触发）：

若用户回复"自动选题"，调用WebSearch获取近期热点，生成主题列表：

> **【自动选题结果】**
>
> 根据近期{行业}热点，为你生成以下主题：
>
> 1. [主题1]
> 2. [主题2]
> 3. [主题3]
> ...
>
> 请确认主题列表，或告诉我需要调整的部分。

### 2. 批量预览

生成所有篇的**标题+首段预览**（每篇约100字）。

**输出格式**：

> **【批量预览】**
>
> **第1篇**：[标题]
> [首段预览，约100字]
>
> **第2篇**：[标题]
> [首段预览，约100字]
>
> ...
>
> 请确认预览，或告诉我需要调整的篇目。

### 3. 一键生成完整正文

**生成规则**：

| 规则 | 说明 |
|------|------|
| 统一框架 | 所有篇使用相同框架（除非用户指定不同） |
| 统一风格 | 所有篇使用相同写作风格 |
| 差异化标题 | 每篇生成3个备选标题 |
| 差异化正文 | 每篇正文根据主题差异化 |

**输出格式**：

> **【批量生成结果】**
>
> **第1篇**：[标题]
> [完整正文]
> **评分**：XX/100
>
> **第2篇**：[标题]
> [完整正文]
> **评分**：XX/100
>
> ...

### 4. 用户选择后续操作

**引导话术**：

> 全部生成完成 🙌 请选择后续操作：
>
> - 回复 **"全部满意"** → 进入批量排版
> - 回复 **"优化第X篇"** → 优化指定篇目
> - 回复 **"换个标题"** → 为所有篇目重新生成标题
> - 回复 **"只保留第X篇"** → 删除其他篇目
> - 回复 **"全部换风格"** → 切换写作风格重新生成

### 5. 批量自动迭代（条件触发）

**触发条件**：用户选择"自动迭代"（质量门控对齐：统一目标为92分输出门槛）

**执行流程**（与质量门控规则保持一致，见 `shared-definitions.md 第十章`）：

1. **逐篇评分**：对每篇内容按评分详情打分
2. **判断达标**：总分 ≥ 92分 → 标记为"通过"，停止迭代
3. **未达标（<92分）→ 判断是否重写**：
   - <80分：自动重写全文
   - 80-91分：针对性优化（保留框架，优化扣分最多的子项）
4. **重写规则**：最多重写2次（初稿 + 重写1 + 重写2）
5. **2次重写后仍未达92分**：输出当前最高分稿 + 说明未达标原因

**迭代内容漂移检测**：每轮迭代后，对比当前内容与用户原始需求的关键词匹配度，匹配度下降超过20%自动回滚。

**自动重写章节映射规则**（同快速模式，见 `sections/quick-mode.md`）

## 批量排版

**执行步骤**：

1. **统一排版偏好**：询问用户排版偏好（或直接使用行业默认）
2. **批量生成排版**：为所有篇目生成排版（HTML + Markdown双格式）
3. **批量合规扫描**：逐篇进行合规敏感词扫描
4. **批量输出**：输出所有篇目的最终文件

**输出格式**：

> **【批量排版结果】**
>
> 已为所有篇目生成排版 🙌
>
> **第1篇**：{文件名}.html + {文件名}.md
> **第2篇**：{文件名}.html + {文件名}.md
> ...
>
> 文件已保存到：{路径}
